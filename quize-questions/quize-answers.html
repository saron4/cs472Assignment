As we know that HTTP and Web Server both are stateless, the only way to maintain a session is when some unique information about the session (session id) is passed between server and client in every request and response.so one way to provide unique id in request and response is using URL rewriting. 12. Why does the request attribute report ‘null’ for the maintaining state demo?

Ans:- A reasonably likely explanation is that the requests are not in the same session. Maintaining a session requires cooperation from the client, which is not guaranteed to be given. 8. What does the operation getJspBody().invoke(null) do when called in a doTag() method?

When getJspBody().invoke(null) is called, it is the output resulting from the execution of the tag body's JSP content that gets passed to the client, not the JSP code itself. Therefore, the doTag method has no way of accessing the tag body output. All it can do is pass it along.
